# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: auth (signup / login / logout), clinic profile setup, and the 8-question plan builder with live break-even preview + draft persist. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Riskiest assumption this UI defends against:** owner trusts the break-even math enough to stay in the builder and publish. Every visual decision below is in service of calm, unsurprising numbers that feel operator-grade, not marketing-grade.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (to be initialized at phase start) |
| Preset | default (New York style, slate base, CSS variables enabled) |
| Component library | Radix Primitives (via shadcn) |
| Icon library | lucide-react 1.8.0 |
| Font | Geist Sans (primary, variable), Geist Mono (numbers in break-even panel only) |

```bash
pnpm dlx shadcn@latest init
# Select: TypeScript → Yes, Style → New York, Base color → Slate, CSS variables → Yes
```

```bash
pnpm dlx shadcn@latest add button input label card form select radio-group \
  slider separator avatar dropdown-menu sonner skeleton
```

No third-party registries in Phase 1. `shadcn view` vetting gate: not applicable.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, form help-text offset |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, input vertical rhythm |
| lg | 24px | Card padding, section gaps within a page |
| xl | 32px | Layout gaps, wizard step separation |
| 2xl | 48px | Major section breaks, auth card internal vertical padding |
| 3xl | 64px | Top nav height, sidebar width multiple (240 = 64×3.75 allowed exception) |

**Exceptions:**
- Sidebar width: **240px** (4px multiple, not on named scale)
- Top nav height: **64px** (= 3xl token)
- Break-even panel min-width: **400px** desktop (4px multiple)
- Auth card max-width: **420px** (4px multiple)
- Wizard content max-width: **640px** (4px multiple) for readable line length
- Logo upload dropzone: **160px** square

**Touch targets:** all interactive elements minimum **40px** height (md default), buttons default **40px**, compact icon buttons **32px** — none below 32px.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Form labels, descriptive text, table cells |
| Body Emphasis | 14px | 600 (semibold) | 1.5 | Inline emphasis, active nav item, form required markers |
| Heading | 20px | 600 (semibold) | 1.3 | Page headings, card titles, wizard step title |
| Display | 28px | 600 (semibold) | 1.2 | Auth screen titles ("Create your clinic account"), major dashboard H1 |

**Numeric display (break-even panel only):** Geist Mono, 14px regular for row values, 20px semibold for tier monthly fee and break-even member count. Tabular figures via `font-variant-numeric: tabular-nums` so columns align.

**Only 4 sizes, 2 weights used across the entire phase.** No italics. No all-caps except the two-char currency symbol on input suffixes.

## Color

PawPlan dashboard palette (the owner's surface — independent of the 6-preset accent palette used on the public enrollment page, which is Phase 3).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FAFAF7` (warm off-white) | Page backgrounds, main content surface |
| Secondary (30%) | `#FFFFFF` (pure white) | Cards, sidebar, top nav, input surfaces |
| Border / divider | `#E8E6E0` (warm gray 200) | Card borders, table rules, input borders at rest |
| Text primary | `#1C1B18` (near-black warm) | All body copy |
| Text secondary | `#6B6A63` (warm gray 500) | Help text, inactive nav, placeholder copy |
| Accent (10%) | `#2F7D6E` (confident sage-teal) | See reserved list below |
| Destructive | `#B4321E` (deep warm red, WCAG-AA on white) | Destructive actions only |
| Success | `#2F7D6E` (reuse accent — same hue family) | Saved-draft confirmation, field-valid checkmark |

**Accent (`#2F7D6E`) reserved for:**
1. Primary CTA button background (`Create account`, `Log in`, `Save draft`)
2. Active sidebar nav item (left 3px rail + text color)
3. Focus ring on inputs (2px outline, 2px offset)
4. Break-even panel "members to break even" number (the single most important figure on screen)
5. Checked state of radio / checkbox / slider thumb

**Accent NOT used for:**
- Link text in body copy (use `Text primary` with underline offset)
- Hover states on secondary buttons (use `Border` darkened)
- Selected form-field borders outside focus (use `Text primary` at 1px)
- Table row hover (use `#F4F2EC` warm gray 100)

**60 / 30 / 10 split confirmed.** No other hues. No brand blue. No medical green. The single accent reads as "trustworthy operator tool" without bleeding into clinical territory.

These are NOT used on the owner dashboard. They appear on Phase 3's public enrollment page and in a 6-swatch picker on the clinic profile page. All values verified WCAG AA (>= 4.5:1) for white text on the swatch and for the swatch itself on white.

## Component Patterns

### Forms (signup, login, profile, builder)
- React Hook Form + Zod resolver. Inline validation fires on blur, re-validates on change after first error.
- Labels above inputs (never floating, never placeholder-as-label). Labels are `Body Emphasis`.
- Helper text below the input in `Text secondary`. Error text replaces helper text, in `Destructive`, prefixed with no icon (keeps layout calm).
- Required fields marked with `*` after label in `Destructive`. Optional fields marked `(optional)` in `Text secondary`.
- Input height: 40px. Padding: 12px horizontal, 8px vertical. Border 1px `Border` at rest, 2px `Text primary` on hover, 2px `Accent` with 2px offset on focus.
- Submit button: 40px height, full-width on auth screens, auto-width on builder/profile.

### Dashboard shell
- **Top nav (64px):** PawPlan wordmark left (18px Geist Sans 600, `Text primary`, followed by "/" separator then clinic name in `Text secondary`), logout button right (ghost button, 32px, `LogOut` icon + "Log out" text).
- **Sidebar (240px, fixed):** nav items 40px tall, 12px padding horizontal, 16px icon (lucide), 8px gap to label. Active item: 3px `Accent` left rail, `Accent` text, `#F4F2EC` background. Items for Phase 1: `Plans` (`LayoutGrid` icon), `Profile` (`Settings` icon).
- **Main content area:** 32px top padding, 48px left padding (24px below 1280px), max-width 1280px, left-aligned.

### Plan builder wizard
- Single-page wizard (not multi-step routes) with 8 question blocks stacked vertically inside the left 60% pane, each block 32px apart.
- Each question = a Card with 24px padding: label on top (`Body Emphasis`), helper text underneath (`Body` in `Text secondary`), control below with 16px gap.
- Question types:
  - **Species mix:** two sliders (dogs %, cats %) with live sum validation, % number shown in `Geist Mono` 14px next to each.
  - **Exam / dental / vaccine prices:** `Input` with `$` prefix adornment, numeric-only, 2 decimal places.
  - **Vaccine cadence:** RadioGroup, 3 options ("Annual", "Every 2 years", "Every 3 years").
  - **Prevention inclusion:** Checkbox group (Heartworm prevention, Flea/tick prevention).
  - **Member discount:** Slider 0–20% with `Geist Mono` value display.
  - **Tier count:** RadioGroup, 2 options ("2 tiers", "3 tiers").
- `Save draft` button: sticky bottom of wizard pane, full-width, `Accent` background. Helper text below: `Autosaves every 30s · Last saved 2 minutes ago` (Body, Text secondary, updates live via react-query).

### Break-even preview panel
- Right 40% pane, sticky (`position: sticky; top: 96px`), min-width 400px, Card with 24px padding, no border — just `#F4F2EC` background to separate from dominant surface.
- Panel heading: `Break-even math` (Heading 20px, semibold). Below: one line body copy — `Updates as you type. Numbers already include Stripe fees and PawPlan's 10% platform fee.` (Body, Text secondary).
- For each tier (2 or 3):
  - Tier name (`Body Emphasis`, 14px) + tier monthly fee (`Geist Mono` 20px semibold, right-aligned). 16px gap below.
  - Line-item table, one row per line, 32px row height, `Geist Mono` 14px values right-aligned:
    - Retail value bundled
    - Monthly fee (member pays)
    - Stripe processing estimate (shown as negative)
    - PawPlan platform fee 10% (shown as negative)
    - Clinic gross per enrolled pet per year
    - **Break-even member count** (rendered in `Accent` color, 20px Geist Mono semibold — this is the emotional payoff number)
  - 24px separator between tiers.
- Empty state (before any inputs): show skeleton rows — never show `0` or `—`, which would read as broken. Skeleton copy: `Fill in the builder to see your numbers.` (Body, Text secondary, centered).

### Profile page
- Single-column form, max-width 720px. Sections: `Practice details` (name, slug — read-only with lock icon tooltip "Slug is locked once created"), `Branding` (logo URL input, 6-swatch accent picker).
- Logo handling (v1): URL input only. Below the input, a 160×160px preview square with `Border` dashed border when empty, rendered logo when URL resolves, error state with `AlertCircle` icon + `That URL didn't load an image.` if broken. Defer native upload to later phase — documented as a known shortcut.
- `Save changes` button: primary, right-aligned, 40px height.

### Draft resume (dashboard empty vs. draft states)
- **Empty state (no draft):** Centered hero, max-width 480px, 32px above fold.
  - Heading: `Build your first wellness plan` (Display, 28px)
  - Body: `Answer 8 questions. See your break-even math update live. Save a draft whenever you want.` (Body, Text secondary)
  - CTA: `Start plan builder` (primary button, `ArrowRight` icon trailing)
- **Draft state:** Card with 24px padding, 480px max-width.
  - Heading: `Plan draft — last edited {relative time}` (Heading, 20px)
  - Body: summary line — `{tierCount} tiers · {speciesMix} · saved 2m ago` (Body, Text secondary)
  - Two buttons: `Resume builder` (primary) + `Delete draft` (ghost, Destructive text)

### Toasts (sonner)
- Position: bottom-right, 24px offset.
- Success toast: `Check` icon in Accent, Body 14px, auto-dismiss 4s.
- Error toast: `AlertCircle` icon in Destructive, Body 14px, auto-dismiss 6s, includes a `Retry` action when applicable.

## Copywriting Contract

Tone: direct, operator-grade, calm. Avoid marketing verbs ("Unleash!", "Supercharge!", "Launch now!"). Avoid clinical jargon. Model: Shopify admin, Linear, Stripe Dashboard.

| Element | Copy |
|---------|------|
| Primary CTA — signup | `Create clinic account` |
| Primary CTA — login | `Log in` |
| Primary CTA — profile save | `Save changes` |
| Primary CTA — builder save | `Save draft` |
| Primary CTA — resume | `Resume builder` |
| Secondary CTA — signup link from login | `Create one` (after `No account yet?`) |
| Secondary CTA — login link from signup | `Log in` (after `Already have an account?`) |
| Signup page H1 | `Create your clinic account` |
| Signup helper under clinic name input | `The public name pet owners will see.` |
| Signup helper under slug input | `This becomes your enrollment page URL. You can't change it later.` |
| Login page H1 | `Log in to PawPlan` |
| Dashboard empty-state heading | `Build your first wellness plan` |
| Dashboard empty-state body | `Answer 8 questions. See your break-even math update live. Save a draft whenever you want.` |
| Profile section helper — slug | `Locked. Contact support to change.` (with `Lock` icon) |
| Profile section helper — logo URL | `Paste a direct link to your logo. PNG or SVG works best.` |
| Profile section helper — accent color | `Pick the color your enrollment page will use.` |
| Builder wizard H1 | `New wellness plan` |
| Builder autosave indicator | `Autosaves every 30s · Last saved {relativeTime}` |
| Builder autosave — saving now | `Saving…` |
| Builder autosave — error | `Couldn't save. Check your connection.` |
| Break-even panel heading | `Break-even math` |
| Break-even panel subtitle | `Updates as you type. Numbers already include Stripe fees and PawPlan's 10% platform fee.` |
| Break-even empty state | `Fill in the builder to see your numbers.` |
| Break-even line labels | `Retail value bundled`, `Monthly fee`, `Stripe processing estimate`, `PawPlan platform fee (10%)`, `Clinic gross per pet per year`, `Members to break even` |
| Error — email taken | `That email is already in use. Log in instead?` (with `Log in` link) |
| Error — bad credentials | `Email or password didn't match. Try again.` |
| Error — slug taken | `That URL is taken. Try another.` |
| Error — slug invalid | `Use only lowercase letters, numbers, and hyphens.` |
| Error — network | `Couldn't reach PawPlan. Check your connection and try again.` |
| Error — server | `Something broke on our end. We're looking into it.` |
| Destructive — delete draft | Confirmation title: `Delete this draft?` · Body: `This can't be undone. Your answers will be lost.` · Confirm button: `Delete draft` (destructive) · Cancel button: `Keep draft` |
| Logout | Button label: `Log out`. No confirmation — low-cost action. |

**Empty states use "Build your first…" pattern, not "No X yet."** Frames the absence as momentum, not failure.

**Errors name the problem + the fix.** Never generic ("Something went wrong.") unless server-side with no known fix.

## Interaction States

Every interactive element declares the following states. Colors refer to the palette above.

| Element | Rest | Hover | Focus | Active | Disabled |
|---------|------|-------|-------|--------|----------|
| Primary button | Accent bg, white text | Accent darken 8% | 2px Accent ring, 2px offset | Accent darken 12% | Accent at 40% opacity, cursor not-allowed |
| Secondary button | White bg, Border 1px, Text primary | `#F4F2EC` bg | 2px Accent ring, 2px offset | `#E8E6E0` bg | 40% opacity |
| Ghost button | transparent, Text primary | `#F4F2EC` bg | 2px Accent ring, 2px offset | `#E8E6E0` bg | 40% opacity |
| Destructive button | Destructive bg, white text | Destructive darken 8% | 2px Destructive ring | Destructive darken 12% | 40% opacity |
| Input | Border 1px at rest | Border darkens to Text primary | 2px Accent ring, 2px offset | — | `#F4F2EC` bg, no border change |
| Nav item | Text secondary | Text primary + `#F4F2EC` bg | 2px Accent ring, 2px inset | 3px Accent rail + Accent text + `#F4F2EC` bg | — |
| Swatch (accent picker) | 48px tile, border 1px Border | Border 2px Text primary | 2px Accent ring, 2px offset | 2px Accent border + Check icon overlay | — |
| Slider | Track Border, thumb 16px white + 1px Border | Thumb border Text primary | 2px Accent ring | Thumb bg Accent | 40% opacity |

**All interactive elements have visible focus rings.** Never rely on browser default, never remove focus outline without a replacement.

## Loading & Skeleton Patterns

- Page-level navigation: no spinner — use Next.js 16 streaming + Suspense with skeleton rows from shadcn `Skeleton`.
- Break-even panel: skeletons are the empty state. When inputs change, the new values animate in via a 150ms crossfade on the value text only — never layout shift.
- Save draft button: on click, label changes `Save draft` → `Saving…` + disabled. On success, toast. On failure, toast + button re-enables.
- Form submission: primary button shows `Loader2` icon spinning + label changes to active verb form (`Creating account…`, `Logging in…`, `Saving…`).

## Accessibility Contract

- All text AA minimum (`#6B6A63` on `#FAFAF7` = 4.6:1 ✓; `#1C1B18` on `#FAFAF7` = 17.2:1 ✓; `#2F7D6E` on white = 5.1:1 ✓; white on `#2F7D6E` = 5.1:1 ✓; `#B4321E` on white = 5.9:1 ✓).
- Keyboard navigable: every action reachable via Tab. Escape closes modals and the delete-draft confirmation. Enter submits forms.
- Screen readers: every form field has a `<label>` association. Every icon-only button has `aria-label`. Error messages use `aria-live="polite"`, autosave indicator uses `aria-live="polite"`. Validation errors linked via `aria-describedby`.
- Reduced motion: respect `prefers-reduced-motion`. The 150ms crossfade on break-even values drops to 0ms. No parallax, no decorative animations.
- Focus management: after submitting signup, focus moves to the dashboard empty-state heading. After saving a draft, focus returns to the `Save draft` button.
